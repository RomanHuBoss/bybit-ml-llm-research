CHANGELOG — Strategy Quality Gate / Research Candidate separation
Date: 2026-04-30

Problem fixed:
- The system could display REVIEW_ENTRY while also saying that the backtest was weak, small, or missing.
- This made raw research setups look like actionable recommendations.

Core changes:
1. Added app/strategy_quality.py
   - Classifies category + symbol + interval + strategy into APPROVED, WATCHLIST, RESEARCH, REJECTED, STALE.
   - Computes quality_score, evidence_grade, reason and diagnostics.

2. Added SQL table strategy_quality
   - Persisted in sql/schema.sql.
   - Indexed by category, interval, status and score.

3. Changed recommendation contract
   - Weak/missing backtest no longer produces REVIEW_ENTRY.
   - Non-approved setups become RESEARCH_CANDIDATE.
   - REVIEW_ENTRY requires APPROVED strategy unless REQUIRE_STRATEGY_APPROVAL_FOR_REVIEW=false.

4. Changed backtest flow
   - app/backtest.py updates strategy_quality after a run.
   - app/backtest_background.py now works in strategy_matrix mode across symbols x intervals x strategies.
   - Default backtest horizon increased to BACKTEST_AUTO_LIMIT=30000.

5. Changed research/API layer
   - app/research.py joins strategy_quality and uses it in ranking.
   - app/api.py exposes /api/strategies/quality and /api/strategies/quality/refresh.
   - /api/status reflects strategy-quality gate settings.

6. Changed frontend
   - Added RESEARCH queue filter.
   - Added RESEARCH_CANDIDATE visual status.
   - Added Quality column in raw table.
   - Added strategy_quality item in checklist.

7. Updated docs/config/tests
   - .env.example includes gate parameters.
   - README includes Revision V19.
   - docs/STRATEGY_QUALITY_GATE_2026-04-30.md explains the new contract.
   - Regression tests updated for the new behavior.
