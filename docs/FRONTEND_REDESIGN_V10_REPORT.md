# Frontend redesign v10 — Bybit Trading Decision Cockpit

## Цель версии

v10 исправляет главный дефект предыдущей перевёрстки: контент панели параметров и кнопки операций могли не помещаться в левую колонку на 1366–1500 px и частично обрезались. Верстка переработана так, чтобы dashboard не создавал горизонтальный overflow: элементы сжимаются, переносятся или переходят в одно-/двухколоночный режим по breakpoint.

## Ключевые изменения верстки

1. **Viewport-safe layout**
   - Уменьшена ширина боковой навигации.
   - Основная сетка теперь использует более безопасные размеры колонок: left 260–320 px, center `minmax(0, 1fr)`, right 280–360 px.
   - На ширине до 1500 px правая аналитика уходит вниз, а центральная рабочая область остается читаемой.
   - На ширине до 1180 px экран становится одноколоночным, а центральный блок сигнала поднимается выше очереди и настроек.

2. **Панель параметров больше не режет контент**
   - В узкой левой колонке `ops-body` переведен в одну колонку.
   - Основные и вторичные кнопки операций больше не стоят в тесной 2-колоночной сетке внутри боковой панели.
   - Для кнопок включены переносы текста и `overflow-wrap`.
   - Select/input ограничены шириной контейнера.

3. **Очередь рекомендаций стала компактнее**
   - Убраны лишние декоративные элементы внутри строки кандидата.
   - Сетка кандидата адаптирована под 260–320 px.
   - Длинные символы/таймфреймы/бейджи получают ellipsis без разрушения сетки.

4. **Главный торговый сигнал стал приоритетом на ноутбуках и планшетах**
   - При переходе в одну колонку порядок блоков: `center-stack` → `right-stack` → `left-stack`.
   - Пользователь сначала видит SignalCard, карту сделки и TradeSheet, а не техническую панель настроек.

5. **Дополнена аналитическая панель**
   - В блок microstructure добавлена строка `Ликвидность / spread`.
   - JS обновляет ее из `liquidity_score` и `spread_pct` без изменения API-контракта.

6. **Таблицы и техническая зона**
   - Горизонтальный scroll локализован внутри `.raw-table-wrap`.
   - Sticky header сохранен.
   - Ячейки получили безопасные ограничения по ширине и ellipsis.

## JS-изменения

- Сохранены текущие endpoints и бизнес-логика.
- Добавлено обновление нового поля `liquidityBox`.
- Сохранены обработчики глобальных JS/Promise ошибок из v9.
- Сохранена сортировка полной таблицы сигналов по заголовкам.

## Проверка

- `node --check frontend/app.js` — пройдено.
- HTML/CSS/JS находятся в исходной структуре проекта: `frontend/index.html`, `frontend/styles.css`, `frontend/app.js`.

## Оставшиеся backend/API-зависимости

- Реальный свечной график требует OHLCV endpoint или подключения TradingView/lightweight-charts.
- Funding, OI, liquidations будут точнее, если backend начнет отдавать эти поля для каждого кандидата.
- Текущая версия не меняет формат API и не отправляет торговые ордера.
