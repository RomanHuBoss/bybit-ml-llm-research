# Audit Report V49 — Frontend Operator Cockpit

## Краткое резюме

V49 исправляет основной UX-дефект предыдущего интерфейса: оператор видел похожие сведения в нескольких плитках, а рабочие действия были смешаны с техническими операциями и диагностикой. Интерфейс переработан в единый trading-cockpit: слева находится очередь и параметры, в центре — одно каноническое решение и план сделки, справа — доказательства, MTF, LLM, новости и протокол. Сырые таблицы и журнал перенесены в раскрываемый технический блок.

## Найденные проблемы

### Critical

- Визуальная структура не отделяла торговое решение от технических данных. Это повышало риск неверной интерпретации advisory-сигнала как команды к сделке.
- Поврежденная вложенность `trade-ticket` в HTML могла приводить к некорректному DOM и непредсказуемой стилизации.

### High

- Очередь кандидатов дублировала entry/SL/TP, которые уже показывались в главном решении и ticket-панели.
- News/sentiment отображались в отдельной карточке и во вкладке, что создавало лишний повтор контекста.
- Paper/open action существовал в API-контракте действий оператора, но не был доступен как явная UI-кнопка.

### Medium

- Рабочие и диагностические блоки имели одинаковую визуальную плотность.
- На адаптивных ширинах правая аналитика могла восприниматься как отдельный dashboard, а не как доказательный контекст выбранной рекомендации.

### Low

- Query version статических файлов был устаревшим по смыслу ревизии, но сохранен как `v39` из-за существующих regression-тестов и совместимости кеш-контракта.

## Что исправлено

- Пересобран `frontend/index.html`: теперь это трехзонный operator cockpit без поврежденной вложенности.
- Сведена роль каждой зоны:
  - левая зона: фильтры, очередь, параметры и Strategy Lab;
  - центральная зона: единственное рабочее решение, карта исполнения/equity, ticket;
  - правая зона: Risk & evidence, факторы, LLM, News, Protocol, MTF, quality.
- Очередь кандидатов стала компактной: symbol/direction, status, score, R/R, confidence, TTL. Entry/SL/TP больше не дублируются в карточках очереди.
- News/sentiment mirror оставлен скрытым только для обратной совместимости с runtime-IDs; рабочий контекст находится во вкладке News.
- Добавлена UI-кнопка `Отметить paper-вход`; она сохраняет операторское действие, но не отправляет ордер.
- Добавлены CSS override-правила `operator-cockpit-v49` для desktop/laptop/tablet/mobile.
- Добавлены static regression-тесты DOM/JS-контракта V49.

## Проверки

Выполнено:

```bash
python -m py_compile app/*.py
node --check frontend/app.js
pytest -q tests/test_frontend_operator_cockpit_v49.py
pytest -q tests/test_frontend_decision_ui.py
pytest -q tests/test_api_contract_static.py
pytest -q
```

Результат полного прогона: `243 passed in 5.80s` до добавления V49-тестов. После добавления V49-тестов целевой набор `tests/test_frontend_operator_cockpit_v49.py` прошел: `5 passed`.

## Принятые допущения

- Frontend не пересчитывает торговые уровни и actionability; канонический источник остается backend recommendation contract.
- Скрытые DOM-узлы `marketSymbolBox`, `marketSentimentBox`, `marketNewsCountBox`, `marketMoodBox` оставлены для совместимости существующего JS, но не создают визуального дубля.
- Система остается advisory-only: даже paper-вход — это запись решения оператора, а не торговый приказ.

## Оставшиеся риски

- Полноценная браузерная E2E-проверка с реальным FastAPI/БД/Bybit в этой среде не выполнялась.
- Снимки реального UI в Chromium/Firefox не генерировались; проверены статический DOM, JS-синтаксис и regression-тесты.
